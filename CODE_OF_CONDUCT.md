# Code of Conduct

## Purpose

u-Segment3D is an open-source scientific software project for bioimage analysis. We want the project to be useful, welcoming, and respectful for users, contributors, maintainers, and collaborators across research disciplines and career stages.

## Expected Behavior

Participants in this project are expected to:

- Be respectful, constructive, and patient in discussions.
- Welcome questions from new users and contributors.
- Give and receive feedback in a professional manner.
- Focus criticism on ideas, code, documentation, and scientific reasoning rather than individuals.
- Respect differences in experience, background, identity, language, institution, and research community.
- Credit prior work and contributions appropriately.
- Help maintain a collaborative environment for reproducible, open scientific software.

## Unacceptable Behavior

Unacceptable behavior includes:

- Harassment, intimidation, threats, or personal attacks.
- Discriminatory comments or behavior related to identity, background, affiliation, career stage, disability, race, ethnicity, nationality, religion, gender, gender identity, sexual orientation, age, or other personal characteristics.
- Sexualized language, imagery, or unwanted attention.
- Deliberate disruption of discussions, issues, pull requests, meetings, or community spaces.
- Public or private sharing of others' private information without permission.
- Repeatedly contacting someone after being asked to stop.
- Misrepresentation of scientific results, authorship, software capabilities, or contributor roles.

## Scope

This Code of Conduct applies to all project spaces, including GitHub repositories, issues, pull requests, discussions, documentation, tutorials, workshops, mailing lists, and project-related community interactions.

## Reporting

For routine technical questions, bug reports, or feature requests, please use the GitHub issue tracker.

For private conduct concerns, contact the project maintainers using the contact information listed in the project README or repository metadata. Reports should include relevant context, links, screenshots, or other details when possible. Maintainers will handle reports with discretion and will limit sharing of sensitive information to what is necessary to evaluate and respond to the concern.

## Enforcement

Maintainers are responsible for clarifying standards of acceptable behavior and may take action when behavior does not meet this Code of Conduct. Possible actions include:

- Clarifying expectations with the participant.
- Asking for edits to comments, issues, pull requests, documentation, or other materials.
- Removing inappropriate content.
- Temporarily or permanently restricting participation in project spaces.
- Escalating concerns to institutional channels when appropriate.

Enforcement decisions should be made fairly, consistently, and with attention to the safety and integrity of the project community.

## Acknowledgment

This Code of Conduct is intended to support open, reproducible, and community-oriented scientific software development.
